# 체험단 블로거 채용 플랫폼 기획 초안

## 목적

- Revu의 탐색 구조를 참고해 블로거가 체험단 캠페인을 빠르게 찾고 신청하는 첫 화면을 만든다.
- 광고주는 캠페인 모집, 리뷰어 매칭, 신청 현황을 한 화면에서 이해할 수 있게 한다.
- 기존 SPA 구조와 역할 값은 유지하고, 블로거는 `candidate`, 광고주는 `recruiter` 맥락으로 해석한다.

## 1차 화면 범위

- 상단 공지, 검색, 주요 메뉴
- 추천 배너 3개
- 빠른 카테고리 필터
- 캠페인 카드 목록
- 선택한 캠페인 상세 패널
- 캠페인 신청 CTA와 신청 준비 상태 메시지
- 광고주 상담 플로팅 안내

## 사용자 흐름

1. 블로거가 검색어 또는 카테고리로 캠페인을 좁힌다.
2. 캠페인 카드를 선택해 제공 내역, 모집 인원, 마감일, 채널 조건을 확인한다.
3. 신청 버튼을 눌러 신청 준비 상태를 확인한다.
4. 광고주는 플로팅 상담 안내 또는 광고문의 메뉴로 유입된다.

## 디자인 방향

- 레포의 디자인 시스템에 맞춰 파란색 계열, 흰색 배경, 8px 카드 반경을 기본으로 한다.
- Revu와 같은 정보 구조를 참고하되 브랜드명, 카피, 이미지, 색 조합은 새 플랫폼으로 재구성한다.
- 모바일에서는 메뉴 접기, 단일 카드 그리드, 상세 패널 하단 배치로 전환한다.

## 구현 메모

- 현재는 더미 데이터 기반 첫 화면이다.
- Firebase 연결 전에도 캠페인 탐색과 신청 CTA 흐름을 확인할 수 있다.
- 실제 DB 연결 시 `docs/data-guide.md`의 체험단 캠페인 이름 사전을 기준으로 맞춘다.
