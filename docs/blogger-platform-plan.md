# 줄서 제품 기획 초안

## 목적

- 오프라인 매장 사장님이 한산한 시간대를 체험단 방문으로 채우고 실제 방문 후기를 빠르게 확보하게 한다.
- 기존 체험단 플랫폼의 노쇼·허위 후기 문제를 사후 패널티가 아니라 QR·GPS 방문 인증으로 줄인다.
- Revu의 탐색 구조는 참고하되, 줄서의 차별점은 `오프라인 방문 특화 × 자동 방문 인증`으로 둔다.

## 워크시트 기반 핵심 정리

- 핵심 고객: 신규 오픈 후 방문 후기가 부족하고 평일 오후처럼 한산한 시간대를 채우지 못하는 맛집·카페·액티비티 매장 사장님
- 문제: 체험단 모집 후 노쇼, 방문 미인증, 예약 슬롯 관리 부담 때문에 제공 비용은 나가고 후기는 쌓이지 않는다.
- 핵심 가치: 예약 슬롯에 맞춰 방문이 인증된 체험단을 자동 매칭해 초기 방문·후기 부족을 해결한다.
- 차별점: 방문 인증 체크인 데이터가 쌓여 체험단 신뢰점수의 기반이 된다.
- 가격 가설: 방문 1건당 5천~1만 원 매칭 수수료 또는 월 5~10만 원 구독

## MVP 핵심 기능

1. 한산 시간대 예약 슬롯 연동 캠페인 등록
2. QR·GPS 방문 인증 체크인
3. 체크인 후 후기 자동 회수·리마인드

## 1차 화면 범위

- 상단 공지, 검색, 주요 메뉴
- 오프라인 방문 인증 중심 추천 배너
- MVP 핵심 기능 3단계 요약
- 빠른 카테고리 필터
- 캠페인 카드 목록
- 선택한 캠페인 상세 패널
- 한산 시간대, 방문 인증, 후기 회수 조건 표시
- 광고주 상담 플로팅 안내

## 사용자 흐름

1. 매장 사장님이 한산 시간대와 제공 내역을 기준으로 캠페인을 등록한다.
2. 체험단은 예약 가능한 슬롯을 보고 신청한다.
3. 선정된 체험단은 예약 시간에 방문해 QR·GPS로 체크인한다.
4. 방문 확인 후 후기 요청과 리마인드가 자동 발송된다.
5. 매장은 방문 확인, 후기 발행, 캠페인 성과를 확인한다.

## 사장님 등록 SaaS 흐름

- 메인 화면의 `사장님이신가요?` 플로팅 CTA에 줄서 로고 아이콘을 넣고, 클릭 시 사장님 전용 등록 화면으로 전환한다.
- 등록 화면은 `사장님 의뢰 → 매장 등록 → 시간대 분석 → 자동 등록 → 추천 수신 → 최종 선택 → 체크인 확인 → 체험 완료` 흐름을 기준으로 한다.
- 사장님이 직접 입력하는 항목은 매장명, 담당자명, 연락처, 지역, 카테고리, 한산 시간대, 제공 내역, 모집 인원, 예약 연동, 캠페인 목표, 선호 체험단 기준이다.
- 자동화 필요 구간은 시간대 분석, 자동 등록, 추천 수신으로 보고, 사장님 결정 구간은 최종 선택으로 둔다.
- 예약 연동과 QR·GPS 체크인은 외부 연동 데이터로 표시하고, 완료 단계는 신뢰 후기 회수로 정리한다.
