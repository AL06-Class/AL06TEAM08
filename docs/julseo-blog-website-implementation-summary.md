## 프로젝트 개요

- 서비스 이름은 `줄서`입니다.
- Revu 스타일의 캠페인 탐색 화면을 참고하되, 핵심 방향은 오프라인 매장 사장님이 한산한 시간대를 체험단 방문으로 채우는 플랫폼입니다.
- 주요 차별점은 `예약 슬롯`, `QR·GPS 방문 인증`, `후기 자동 회수`, `사장님 체험단 모집 등록`입니다.
- 현재 구현은 React, TypeScript, Vite 기반 SPA 프론트엔드 MVP입니다.

## 구현된 화면

### 1. 메인 캠페인 탐색 화면

- 상단 공지, 줄서 로고, 검색창, 주요 메뉴
- 오프라인 방문 인증 중심의 추천 배너 3개
- MVP 핵심 기능 요약
  - 예약 슬롯 등록
  - QR·GPS 체크인
  - 후기 자동 회수
- 빠른 카테고리 필터
- 체험단 캠페인 카드 목록
- 선택한 캠페인 상세 패널
- 캠페인 신청 상태 메시지
- 우측 하단 `사장님이신가요?` 플로팅 CTA

### 2. 사장님 체험단 모집 등록 화면

- 접속 주소: `http://127.0.0.1:5173/#owner-registration`
- 메인 화면의 플로팅 CTA 또는 상세 패널의 `매장 캠페인 등록 문의` 버튼을 누르면 이동합니다.
- 사장님이 체험단 모집을 등록하는 SaaS형 입력 화면입니다.
- 입력 항목:
  - 매장명
  - 사장님 성함
  - 연락처
  - 지역
  - 카테고리
  - 예약 연동 방식
  - 한산 시간대
  - 모집 인원
  - 제공 내역
  - 캠페인 목표
  - 자동 등록 사용 여부
  - 선호 체험단 기준
- 등록 버튼을 누르면 화면 안에서 캠페인 등록 초안 생성 완료 메시지가 표시됩니다.

## 반영한 체험단 운영 프로세스

첨부된 실제 체험단 프로세스 이미지를 기준으로 아래 흐름을 화면에 반영했습니다.

1. 사장님 의뢰
2. 매장 등록
3. 시간대 분석
4. 자동 등록
5. 추천 수신
6. 최종 선택
7. 체크인 확인
8. 체험 완료

프로세스 색상 의미:

- 자동화 필요: 시간대 분석, 자동 등록, 추천 수신
- 사장님 결정: 사장님 의뢰, 최종 선택
- 외부 연동 데이터: 매장 등록, 체크인 확인
- 완료: 체험 완료, 신뢰 후기 회수

## 데이터 기준

사장님 등록 화면은 추후 Firebase 연결 시 아래 데이터 구조로 이어지도록 정리했습니다.

- 신규 초안 컬렉션: `ownerCampaignDrafts`
- 캠페인 본 데이터: `bloggerCampaigns`
- 예약 슬롯: `reservationSlots`
- 방문 인증: `visitCheckIns`
- 후기 회수: `campaignReviews`, `reviewReminders`

현재는 백엔드 저장 없이 프론트엔드 상태로 등록 완료 메시지만 보여주는 MVP 상태입니다.
