# 줄서 디자인 시스템

## 브랜드

- 서비스명: 줄서 (Julseo)
- 로고: 워드마크 "줄서" + 말풍선/체크 아이콘 조합
- 포인트 컬러: 블루 계열 (버튼, 활성 탭, 강조 텍스트에 사용)
- 배경: 화이트 기본, 콘텐츠는 카드형(둥근 모서리) 레이아웃

## 실제 화면 기준 디자인 방향

홈 화면과 캠페인 등록 폼 화면은 밝은 회청색 페이지 배경 위에 흰색 카드와 블루 액센트를 얹는 방향으로 통일한다.

- 페이지 배경: 아주 옅은 회청색 계열을 기본으로 사용해 흰색 카드가 분리되어 보이게 한다.
- 콘텐츠 폭: 데스크톱 기준 중앙 정렬 컨테이너를 사용하고, 홈 화면은 넓은 카드 그리드, 입력 화면은 폼 영역과 우측 요약 패널의 2컬럼 구조를 사용한다.
- 카드: 흰색 배경, 얇은 블루 그레이 보더, 8px 이하 둥근 모서리, 약한 그림자만 사용한다.
- 타이포그래피: 제목은 굵고 선명하게, 설명문은 작은 크기와 낮은 대비로 계층을 분리한다.
- 버튼과 활성 상태: 주요 CTA와 활성 탭은 진한 블루 배경, 보조 버튼은 흰색 배경+블루 보더를 사용한다.
- 이미지 사용: 홈 화면의 핵심 메시지는 실제 오프라인 매장, 방문 인증, 데이터 대시보드 이미지를 어두운 오버레이와 함께 사용한다.
- 여백: 섹션 간 간격을 충분히 두되, 카드 내부는 업무형 서비스처럼 촘촘하고 정돈된 밀도를 유지한다.

## 상태·의미 색상 규칙

전체 화면에 공통 적용하며, 프로세스 다이어그램과 동일한 규칙을 사용한다.

- 코랄/레드: 자동화가 필요한 구간 (예: 캠페인 자동등록, AI 추천 수신)
- 앰버/오렌지: 사장님이 직접 결정하는 구간 (예: 최종 선택)
- 블루: 외부 연동 데이터 (예: 예약 연동, 검증 프로필)
- 틸/그린: 완료 상태 (예: 체크인 확인, 신뢰 후기 확보)

이 규칙은 대시보드 배지, 프로세스 요약 패널, 폼 하단 태그칩 등 모든 화면에서 동일하게 사용한다.

## 내비게이션 구조 (IA)

- 최상단: 검색바(매장, 지역, 한산 시간대), 로그인, 회원가입, 광고문의(CTA)
- 메인 메뉴: 홈 / 캠페인 / 예약슬롯 / 방문인증 / 후기회수 / 오늘오픈 / 파일럿 / 광고문의
- 이 메뉴 구조는 핵심 기능 3가지(예약슬롯, 방문인증, 후기회수)를 그대로 1뎁스 메뉴로 노출한 것이므로, 향후 화면 설계 시 이 구조를 기준으로 삼는다.

## 홈 화면 구성 패턴

1. 히어로 배너 3종 (각각 이미지+배지+제목+설명)
   - "오프라인 방문 특화" 배지 - 한산 시간대를 방문 후기로 채우기
   - "자동 방문 인증" 배지 - QR·GPS 체크인으로 노쇼 감소
   - "후기 회수" 배지 - 체크인 후 후기 요청 자동화
2. 3단계 이용 순서 카드
   - 01 예약슬롯등록
   - 02 QR·GPS체크인
   - 03 후기자동회수
3. 카테고리 아이콘 목록(원형)
   - 오늘오픈
   - 한산시간
   - 맛집
   - 카페
   - 액티비티
   - 문화
   - 성수팝업
   - 광주파일럿
   - 부천파일럿
   - 운영가이드
4. 캠페인 리스트 + 필터 탭
   - 전체
   - 한산시간
   - 맛집
   - 카페
   - 액티비티
   - 문화
5. 우측 플로팅 CTA 카드
   - "사장님이신가요?"
   - 다크 배경
   - 서비스 요약 + 모집 등록 유도 링크

성수팝업, 광주파일럿, 부천파일럿은 PRD의 1차 타깃 지역 3곳과 대응한다.

### 홈 화면 세부 규칙

- 상단 공지 바는 페이지 최상단에 얇게 배치하고, MVP 반영 사항처럼 운영상 중요한 메시지만 짧게 노출한다.
- 헤더는 로고, 검색바, 로그인, 회원가입, 광고문의 링크를 한 줄에 배치한다.
- 검색바는 둥근 캡슐형으로 만들고, "매장, 지역, 한산 시간대"처럼 검색 범위를 명확히 보여준다.
- 메인 메뉴는 텍스트 탭으로 구성하며, 현재 위치는 블루 텍스트나 하단 강조선으로 표시한다.
- 히어로 배너는 3열 카드 그리드로 배치하고, 이미지 위에 어두운 오버레이를 깔아 흰색 텍스트 가독성을 확보한다.
- 히어로 배너 텍스트는 배지, 큰 제목, 짧은 설명 순서로 배치한다.
- 이용 순서 카드는 3열로 배치하고, 왼쪽 상단에 원형 번호 배지를 둔다.
- 카테고리 아이콘은 원형 썸네일 이미지와 짧은 라벨 조합을 사용한다. 선택 상태는 블루 링으로 표시한다.
- 캠페인 리스트 상단에는 섹션 라벨, 큰 제목, 필터 탭, 캠페인 개수를 함께 보여준다.
- 우측 플로팅 CTA 카드는 다크 배경으로 처리해 사장님 모집 등록 행동을 강하게 유도한다.

## 폼 화면 구성 패턴

캠페인 등록 등 입력 화면 공통 패턴이다.

- 좌측: 섹션 번호 + 섹션 제목 + 부제(다음 단계 안내 문구) → 입력 폼
- 입력 폼: 2컬럼 그리드, 라벨은 입력창 위, placeholder로 예시 텍스트 제공
- 체크박스형 옵션은 제목+설명 조합으로 카드처럼 표시 (예: "시간대 분석 후 자동 등록 사용")
- 하단에 관련 태그칩 노출
  - 추천수신
  - 최종선택
  - QR·GPS체크인
  - 리마인드발송
  - 신뢰후기
- 우측: 전체 프로세스 요약 패널을 사이드에 고정 노출하여, 지금 입력 중인 단계가 전체 흐름 중 어디인지 실시간으로 보여준다.
- 프로세스 요약 패널 8단계
  - 사장님의뢰
  - 매장등록
  - 시간대분석
  - 자동등록
  - 추천수신
  - 최종선택
  - 체크인확인
  - 체험완료
- 프로세스 요약 패널 하단에는 상태 색상 범례를 포함한다.

### 캠페인 등록 폼 세부 규칙

- 화면 전체는 좌측 대형 폼 카드와 우측 프로세스 요약 카드로 나눈다.
- 좌측 폼 카드는 현재 단계 번호, 단계명, 큰 제목, 보조 설명, 섹션 제목, 입력 필드 순서로 구성한다.
- 기본 입력 필드는 2컬럼 그리드로 배치한다. 긴 입력인 제공 내역, 캠페인 목표, 선호 체험단 기준은 전체 폭을 사용한다.
- 라벨은 입력창 위에 작고 굵게 표시하며, placeholder는 실제 입력 예시를 제공한다.
- select 입력은 일반 input과 동일한 높이와 보더를 유지한다.
- focus 상태는 진한 블루 보더와 얇은 포커스 링으로 표시한다.
- 체크박스 옵션은 카드형 행으로 처리하고, 체크박스 옆에 제목과 설명을 함께 보여준다.
- 하단 태그칩은 추천수신, 최종선택, QR·GPS 체크인, 리마인드 발송, 신뢰 후기처럼 전체 프로세스의 핵심 액션을 요약한다.
- 우측 프로세스 요약 패널은 실제 체험단 프로세스를 8단계 카드로 보여준다.
- 프로세스 단계 카드는 상태·의미 색상 규칙의 보더와 배경을 적용한다.
- 범례는 2열 칩 형태로 배치하고, 자동화 필요, 사장님 결정, 외부 연동, 완료를 한눈에 비교할 수 있게 한다.
- 최종 제출 CTA는 폼 하단에 전체 폭 블루 버튼으로 배치한다.
